# 知食角小程序开发规则

## 0. 最高开发宪法（安全、跨端与边界）

本项目已进入**正式上线运行期**，AI 在辅助开发时必须死守以下三条底线及分工：

1. **极致安全感（环境隔离与兜底）**
   - **底线**：绝对禁止直接在 Prod 正式库开发调试！涉及核心数据的修改，必须提供“手动强制锁死开关”和“版本一键回退预案”。
   - **分工**：用户负责拍板确认“数据可以动”并在控制台做物理操作；AI 只负责写“幂等、可重复执行、有防错保护”的代码，绝不私自执行 `db.remove()`。
2. **极度防踩坑（跨端友好）**
   - **底线**：任何架构与配置方案（如多环境隔离）必须为未来跨端（iOS/Android，如 uni-app/RN）留有余地。采用“单代码库+本地配置项”的业界标准，禁止使用“多物理文件夹”等不可扩展的死胡同设计。
3. **极度直白可控（拒绝黑盒）**
   - **底线**：拒绝黑盒魔法。改了哪行代码、微信是怎么物理化上传的、环境是怎么切换的，AI 必须用大白话和图解明明白白地解释清楚。
   - **分工**：用户主导方向与业务验收；AI 负责梳理运行原理、提前列出操作后果，并提供自动化执行脚本（如 PowerShell 大扫除脚本）。

> ⚠️ 警告：只有在满足以上三条底线的前提下，才允许进行功能代码的编写与修改。

## 1. PRD 与计划为准

- 每次执行 task 严格按照既定计划（如 `post-launch-safe-iteration-plan.md` 或 `v2-auth-db-plan.md`）推进。
- 有新想法、方案或重大架构改动，必须先出面向小白的“白话图解计划”，与用户达成一致后再执行。
- 不允许擅自偏离 PRD 范围。

## 2. 小步快跑

- 每次只做计划中的一项，或一项中的一小部分。
- 步步优化，处处闭环，确保开发完整性。
- 每个阶段完成后对照 PRD 验收用例自测。
- **强制通道**：任何待上线功能，必须先通过 Test 库体验版的强制回归验证，方可转 Prod 发布。

## 3. 设计参照

- 涉及前端设计参照 `design/` 中的设计图（花钱买的，必须用好）。
- 确保生成的前端效果与设计严格一致。
