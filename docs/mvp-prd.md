# 《识材》微信小程序 MVP PRD（可快速 Demo）

## 1. 目标

- 跑通闭环：拍照/上传 → 识别结果 → 鲜度判断 → 做法推荐 → 佐料采购单
- 用最小成本验证：用户是否愿意在菜市场使用「拍一下就给出做法与佐料」

## 2. MVP 范围（P0）

### 2.1 页面

- 首页（拍照/上传）
- 结果页（食材名称 + 鲜度结论 + 推荐做法）
- 采购单页（佐料清单勾选 + 一键分享）

### 2.2 功能

- 拍照/相册上传图片
- 启动时建立登录态（静默）：获取/生成 userId（openid 或匿名 id）并本地持久化，用于基础统计、限流与后续扩展
- 单次分析返回结构化结果（见接口约定）
- 结果页选择一个做法并生成采购单
- 采购单勾选与微信原生分享小程序卡片
- 提供「退出登录/清除本地数据」入口：清空本地缓存并重新建立会话（不涉及昵称头像/手机号等完整账号体系）

## 3. 暂不做（MVP Out of Scope）

- 识别准确率/耗时的硬指标承诺与优化工程
- 英文名输出、地区特色菜匹配、成品图版权与检索
- 鲜度形态识别（整条/切块/剥壳）与置信度百分比
- 采购单按摊位顺序分组、海报/小程序码定制生成
- 完整账号体系（昵称头像、手机号绑定、账号合并）、收藏、历史记录、运营后台

## 4. 用户主路径

1. 打开小程序 → 静默登录（建立会话/拿到 userId）→ 进入首页
2. 点击拍照/上传
3. 进入加载态（文案可趣味化）→ 获取结构化分析结果
4. 展示：食材名/简介、鲜度档位与依据、2-3 个做法
5. 选择做法 → 生成佐料采购单
6. 采购单勾选 → 分享给家人/朋友 → 再拍一次

## 5. 关键数据结构（概念）

分析结果对象：

- ingredient\_name：食材中文名
- ingredient\_desc：一句话简介/提醒
- freshness\_level：新鲜｜一般｜不太新鲜
- freshness\_reason：1 句依据
- recipes：做法列表（2-3 个）
  - recipe\_name：做法名
  - ingredients\_needed：佐料数组（字符串）

## 5.1 登录与退出（P0 口径）

目标：不做复杂账号体系，但必须有稳定的「用户会话标识」与「清除本地数据」能力，保证后续可扩展（历史/收藏/配额等）。

- 登录态定义：小程序启动后获得 `userId`，优先使用云端可提供的 openid；无法获取时降级为匿名 `userId`（本地生成并持久化）
- 登录方式：静默，不弹出昵称头像授权弹窗；不依赖 `getUserProfile`
- 退出定义：清空本地缓存（含 `userId`/token 与业务缓存），回到“未登录态”，随后在下一次启动/下一次需要请求时重新建立会话

建议本地存储键（实现口径）：

- `pf_user_id`: string
- `pf_auth_source`: "cloud\_openid" | "wx\_login" | "anonymous"
- `pf_cache_*`: 业务缓存（如有）

## 6. 验收标准（MVP）

- 不开启云开发也能跑通页面闭环（Mock 数据）
- 开启云开发后可通过云函数返回结构化结果并正常渲染
- 冷启动时可自动建立登录态并获得 `userId`（云开发优先 openid；无云开发则匿名 id）
- 执行「退出登录/清除本地数据」后，本地缓存被清空且不影响继续使用核心闭环（可重新建立会话）
- 选择做法后可生成采购单并勾选
- 分享按钮可正常转发小程序卡片

### 7.2 主流程（状态机/页面流转）

```text
[启动]
  ├─ 静默登录 -> 获取/生成 userId -> 写入本地存储
  └─ 失败兜底：允许进入首页（使用匿名 userId），不阻断主流程

[首页]
  ├─ 点击「拍照」/「相册上传」
  └─ 拿到图片 -> 上传/分析 -> 跳转结果页（携带 analysisResult）
              ├─ 成功 -> [结果页]
              └─ 失败 -> 回到首页（Toast：识别失败/网络错误/请重试）

[结果页]
  ├─ 展示：食材名/简介、鲜度档位与依据、做法列表（2-3）
  ├─ 选择一个做法（默认选中第 1 个）
  └─ 点击「生成采购单」-> [采购单页]（携带 recipe + ingredient_name）

[采购单页]
  ├─ 展示：做法名、佐料清单（可勾选）
  ├─ 点击「分享」-> 微信原生分享小程序卡片
  ├─ 点击「更多/设置」（可选入口）-> 退出登录/清除本地数据
  └─ 点击「再拍一次」-> 回到首页
```

    },
    {
      "recipe_name": "红烧鲈鱼",
      "ingredients_needed": ["葱", "姜", "蒜", "酱油", "糖"]
    }
  ]

#### 7.4.2 关键交互与状态

```text
App（全局）
  state:
    - userId: string
    - authSource: "cloud_openid" | "wx_login" | "anonymous"
  action:
    - initAuth() -> set userId/authSource
    - logout() -> clear storage -> initAuth()

首页
  state:
    - imageLocalPath?: string
    - isLoading: boolean
  action:
    - chooseImage(camera|album) -> set imageLocalPath
    - analyze(imageLocalPath) -> result | error

结果页
  state:
    - analysisResult: object
    - selectedRecipeIndex: number（默认 0）
  action:
    - selectRecipe(index) -> set selectedRecipeIndex
    - createList() -> navigate to 采购单页（传 selected recipe）

采购单页
  state:
    - recipe_name: string
    - ingredient_name: string
    - ingredients_needed: string[]
    - checkedMap: { [item: string]: boolean }（默认全选或默认全不选二选一）
  action:
    - toggleItem(item) -> update checkedMap
    - share() -> 微信分享（卡片内容用食材名+做法名）
    - retry() -> back to 首页
```

### 7.5 MVP 验收用例清单（可直接对照）

```text
用例 00：冷启动静默登录
  Given：首次打开小程序
  When：进入首页前完成初始化
  Then：本地存在 userId（openid 或匿名 id），不弹授权弹窗

用例 01：拍照识别成功
  Given：进入首页
  When：拍照并确认
  Then：出现加载态 -> 进入结果页 -> 显示食材名/鲜度/做法列表

用例 02：相册上传识别成功
  When：选择相册图片
  Then：同用例 01

用例 03：结果页生成采购单
  Given：结果页有 2-3 个做法
  When：选择做法 2 并点击「生成佐料采购单」
  Then：进入采购单页 -> 列表与做法 2 匹配

用例 04：采购单勾选 + 分享
  When：取消勾选 1-2 项 -> 点击分享
  Then：触发微信分享面板（卡片正常转发）

用例 05：失败兜底
  When：分析接口失败/超时
  Then：Toast 提示 + 可重试 + 不崩溃

用例 06：退出登录/清除本地数据
  Given：已完成一次识别流程
  When：点击「退出登录/清除本地数据」
  Then：本地缓存被清空 -> 可重新建立会话 -> 核心闭环仍可继续使用
```
