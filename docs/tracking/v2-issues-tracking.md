# 识为鲜 V2 版本问题追踪与分类整理 (V2 Issues Tracking)

## 一、核心业务逻辑与底层数据架构 (Product Logic & Data Architecture)
*优先级：最高（决定产品骨架和数据一致性）*

*   **[问题 10] 核心模块关系梳理**：明确逐道菜选购、汇总选购和历史记录在UI层和底层数据存储层的关系。
*   **[问题 8] 业务闭环与CRUD管理**：明确规划与执行的多对一关系，在一次购买活动闭环后，允许用户按“每道菜”、“每个食材”、“每次购买”为单位进行增删改查。
*   **[问题 5] 历史记录视角拆分**：历史记录模块需区分“逐道菜选购”（按每道菜）和“汇总选购”（按每次购买），与上述底层逻辑对齐。

## 二、核心交互与UI重构 (UI & UX Design Optimization)
*优先级：高（决定用户第一印象与核心操作效率）*

*   **[问题 2] 首页UI平衡与重构**：解决首页两条主线平分秋色导致的重点不突出问题，进行重新设计。
*   **[问题 3] 输入交互扁平化**：取消语音和文本输入的二级入口，改为左右滑动切换（左文本、右语音），缩短高频操作路径。
*   **[问题 7] 逐道菜选购排版优化**：修复规划页面搜索做法区域重叠的问题。
*   **[问题 4] Icon 替换**：更换清单页Icon，解决与首页Icon重复的问题。

## 三、功能完善与体验增强 (Feature Enhancements)
*优先级：中高（提升用户在核心流程中的掌控感）*

*   **[问题 6] 列表项管理权**：逐道菜选购和汇总选购的 items 必须允许用户增删。
*   **[问题 9] 智能防重提示**：落实相似食材的去重提示功能，避免重复购买。

## 四、技术基建与Bug修复 (Technical & Bug Fixes)
*优先级：高（阻塞性问题）*

*   **[问题 1] 语音识别转录修复**：排查语音无法识别转录的原因，明确使用的模型和API接口（微信原生、腾讯云或第三方等），并进行修复。
*   **[问题 11] 集合路由显式化重构（下期）**：`miniprogram/utils/db.js` 当前通过覆盖 `db.collection` 实现匿名用户到 `temp_` 集合的隐式路由。该改造有收益，但属于结构性重构；为避免影响本期小程序过审与上架稳定性，延期到下期处理。本期保持现状，只记录风险与改造方向。
