# 《知食角》微信小程序 V2.0 PRD（双引擎版）

> **最新版本**：v2.0-Draft
> **产品定位**：从单次使用的“食材识别工具”，升级为高频闭环的“识物百科 + 智能购物管家”双引擎生活方式 APP。
> **前置依赖**：基于 V1.0 (MVP) 已跑通的识物、鲜度、做法及双语多文化框架。

## 1. 升级背景与核心目标

V1.0 MVP 验证了“拍照 -> 识物 -> 鲜度 -> 做法 -> 采购单”的单线工具流。
V2.0 将通过引入\*\*“智能语音/文本购物清单”**作为第二条主线，将低频的“猎奇识物”与高频的“买菜规划”深度交织。通过**“一套数据，多个视角”\*\*的先进设计理念，覆盖更广泛的用户心智，打造极深的产品护城河。

### 1.1 核心人群与核心痛点（Aha Moment）

1. **留学生（生存与成长）**：扫一扫认识国外超市的奇特食材，并一键生成/翻译购物清单。
2. **主妇/夫（效率与防坑）**：键盘输入或语音一句话生成结构化购物清单。
3. **逛集市的游客（猎奇与记录）**：将扫过的当地特色食材沉淀为精美的“食材图鉴”，满足收集癖与分享欲。
4. **美食爱好者（专业与极致）**：通过“按菜谱”规划一桌大餐，食材不重复不遗漏，知识深度够（目前mvp-prd.md的知识深度够了）。

## 2. 核心功能模块设计（双主线 & 双视角）

### 2.1 首页：情境自适应架构与 UI 融合 (Context-Aware UI)

**核心原则：不破坏 MVP 既有设计资产（参照 design 文件夹），采用渐进式增强（Progressive Enhancement）。**
首页保留 MVP 经过验证的“重输入”极简骨架，平滑引入新功能：

- **核心交互区（融合设计）**：保留原有的“📷 拍照识食材 / 相册上传”核心视觉锚点。在不破坏原视觉平衡的前提下，新增并列或悬浮的 **“🎤 记一笔”**（语音/文本快捷输入清单）主操作入口，确保两套引擎视觉统一。
- **动态情境展示**：
  - **默认态**：展示当前的“未完成清单卡片”（按区域折叠）。若为空，则展示高品质当季食材推荐。
  - **探索态（刚识物完毕）**：在结果页弹出“一键加入购物清单”快捷操作，并利用 AI 推荐相关佐料（例：加入“排骨”时，AI 提示“是否顺便补充葱姜蒜？”）。
  - **规划态（刚记完清单）**：清单食材旁展示微小的“💡 新鲜度指南”入口，提前告知如何挑选。

### 2.2 购物清单：“规划”与“执行”的双视角

解决用户在“家里构思”和“超市采购”时的认知错位。

- **视角 A：逐道菜选购（Planning View - 按菜品/食谱）**
  - **场景**：在家中或工位上构思吃什么。
  - **呈现**：按菜品分组（Grouped List）。例如：【糖醋排骨】下辖排骨、冰糖、老抽；【蒜蓉生菜】下辖生菜、大蒜。
  - **操作**：支持整道菜的一键删除/撤销；支持添加到购物清单，可选择，但默认加到当天最新的那个购物清单
- **视角 B：汇总选购（Execution View - 逛超市模式）**
  - **场景**：推着购物车在超市里。
  - **呈现**：一键 Toggle 切换至“🛒 逛超市模式”。AI 将所有菜品的食材进行**智能去重合并**（如两道菜都需要的大蒜合并数量），并按超市物理动线（生鲜区、肉类海鲜区、调料干货区）重新分组。
  - **操作**：丝滑的“划线打勾”解压动画。
- **气泡式弱提示合并（交互亮点）**：
  - 当用户分两次录入相似食材（如“猪肉”和“五花肉”）时，系统不静默强制合并，而是在两者之间显示轻量的“气泡连线”。
  - 点击气泡，弹出确认框：“AI 发现 \[猪肉] 和 \[五花肉] 可能是同一类计划，合并为 \[五花肉 800g] 还是 \[保持分开]？”

### 2.3 历史数据：“集邮”与“购物”的双视角

复用同一套底层用户行为日志，满足不同心境诉求。

- **视角 A：集邮视角（探索与成就感）**
  - **场景**：闲暇时光、旅游回味、社交分享。
  - **呈现**：类似精美的“食材图鉴”（网格瀑布流）。不仅包含拍照识物的视觉记录（`sourceType: 'vision'`），也支持通过“记一笔”直接保存的纯文本食谱记录（`sourceType: 'text'`）。按食材种类（菌菇、海鲜等）或地域分类。
  - **状态**：已扫过的食材高亮点亮，未扫过的显示剪影。支持一键生成“我的食材集邮海报”。支持进入编辑模式进行重命名、批量删除等高级操作。
- **视角 B：购物视角（效率与回溯）**
  - **场景**：去超市前不知道买什么、需要补货。
  - **呈现**：高效的账单式列表（List View）。按时间倒序展示历史采购清单。
  - **操作**：提供“一键复购”功能，将上周的常备菜（葱姜蒜、鸡蛋等）迅速加入今天的当前清单中。

### 2.4 MVP 核心功能资产严格继承 (防退化机制)

V2.0 的建设绝不是“推倒重来”，必须 100% 兼容并保留 MVP 验证成功的核心“A-ha Moment”：

1. **跨文化双轨食谱体验**：结果页必须完整保留 `[🍴 熟悉的滋味]` / `[📍 试试当地做法]` 的双 Tab 切换机制，不可降级。
2. **视频和网站教程提取**：V2的“逐道菜选购”清单中，针对具体菜谱下的佐料单，依然需要保留一键“获取视频做法”及缩略图展示的入口。
3. **多语言 (i18n) 与国籍设置**：MVP 跑通的全局中英双语切换（及底层语料库）、国籍定位强制拦截必须平滑继承。
4. **一键社交分享**：保留生成带动态标题的微信原生分享卡片功能。

## 3. V2.0 阶段范围划分 (开发优先级)

为了保证敏捷迭代，我们将 V2.0 拆分为两个阶段：

### P0 (V2.0 核心主干 - 必须实现)

1. **基础“记一笔”功能**：支持纯文本/语音输入（通过小程序直接录音并调用大模型API进行语音转文字），调用大模型提取为结构化食材清单，支持加入当天最新清单，**并同步记录至集邮册（`histories` 表，标记为文本来源）**。
2. **当前购物清单双视角**：实现“按菜品（分组）”与“逛超市（合并按区域）”的 Toggle 切换。
3. **首页 UI 重构**：实现底部双核心按钮（扫一扫、记一笔）以及上方动态清单卡片。
4. **弱提示合并交互**：AI 识别相似项并在前端渲染“可合并”提示，支持用户手动确认。
5. **识物结果的无缝衔接**：支持从识物结果页/菜谱页将食材/佐料一键添加至当天最新的购物清单中。

### P1 (V2.1 增强体验 - 后续迭代)

1. **历史双视角重构**：将现有的“历史记录”升级为“图鉴模式（集邮）”和“账单模式（复购）”。
2. **场景交叉推荐**：识物结果页的“智能佐料推荐”加入清单。
3. **丝滑打勾动效与解压音效**。

## 4. 关键数据结构演进 (概念)

为了支持“双视角”与“自动合并”，数据结构需从单维度的字符串数组升级为包含元数据的实体对象。并且需要引入“清单(List)”的概念。

基于 NoSQL 文档内嵌特性，废弃独立的食材表，将食材列表作为数组（`items`）内嵌到购物清单集合中，并统一采用小驼峰命名（camelCase）。

**Shopping List Entity (购物清单表，含内嵌食材 items)：**

```json
{
  "_id": "list_20260403_01",
  "_openid": "wx_openid_xxx",
  "title": "2026-04-03 购物清单", // 默认名称，按天生成
  "status": "active", // active, completed
  "items": [
    {
      "id": "ing_123",
      "name": "排骨",
      "standardName": "猪排骨", // 用于 AI 归一化去重
      "quantity": 500,
      "unit": "g",
      "category": "鱼鲜肉蛋", // 超市动线分类
      "sourceRecipe": "糖醋排骨", // 来源菜谱，用于逐道菜选购分组，若为直接语音/键盘输入则为空或 "直接添加"
      "status": "pending", // pending, bought
      "createdAt": "2026-04-03T10:00:00Z"
    }
  ],
  "createdAt": "2026-04-03T09:00:00Z",
  "updatedAt": "2026-04-03T09:00:00Z"
}
```

**History Entity (历史记录/集邮册)：**
新增 `sourceType` 区分来源，支持文本/语音输入的食谱收藏。

```json
{
  "_id": "history_id_xxx",
  "_openid": "wx_openid_xxx",
  "sourceType": "text", // vision (拍照) | text (文本/语音)
  "ingredientName": "排骨", // 文本食谱可为空
  "selectedRecipe": {
    "recipeName": "糖醋排骨",
    "ingredientsNeeded": [...]
  },
  "analysisResult": { ... }, // 列表查询时需过滤，纯文本可为 null
  "cloudImagePath": "", // 文本输入为空
  "createdAt": "2026-04-03T09:00:00Z",
  "updatedAt": "2026-04-03T09:00:00Z"
}
```

**Shopping List State (前端清单状态推导)：**

- 前端通过对当前活跃清单（`_id`）的 `items` 数组，按 `sourceRecipe` 进行 `groupBy`，即可渲染**“逐道菜选购（按菜谱）”**。
- 前端通过对 `category` 和 `standardName` 进行 `groupBy` 和数量累加，即可渲染**“汇总选购（按动线）”**。

## 5. 验收标准预告

1. **多端点汇聚清单**：无论从首页“记一笔”语音/键盘输入，还是从识物结果的“菜谱”中点击添加，都能默认汇聚到当天最新的购物清单中。
2. **结构化提取**：用户键盘输入或说出“我要买两斤排骨和一头蒜做糖醋排骨，再买点白菜”，系统能准确生成含数量、单位和来源菜谱的 3 个食材项。
3. **双视角切换**：在当天清单中切换至“逛超市模式”时，排骨归入“肉类区”，白菜和蒜归入“蔬菜区”，并且跨菜谱的相同食材能正确累加数量。
4. **智能合并提示**：先后录入“大葱”和“小葱”，列表能出现“气泡合并提示”，且用户点击合并后，原先的两项变为合并后的一项。
5. **历史记录与高级操作**：“集邮”页面能正常显示过去识物（`vision`）和记一笔（`text`）的图片/卡片流，支持批量删除和重命名；“购物”视角能按时间倒序清晰展示历史清单，且列表查询性能优异无卡顿。
