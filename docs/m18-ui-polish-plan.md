# M18: 国际化翻译与排版精细打磨计划 (UI Polish Plan)

## 背景

在完成了基础的国际化架构 (M13-M17) 后，针对英文文案的地道性、准确性以及由此引发的 UI 排版细节问题进行逐页走查与优化。

## 工作流

1. **收集截图**：用户将各页面的中英文截图存入 `docs/screenshots/m18-i18n-review/`，并提供对照关系。
2. **诊断分析**：逐页分析视觉排版（折行、间距、溢出）与文案质量（是否符合母语习惯，是否具备引导性）。
3. **执行优化**：修改 `en.js` 字典文件，必要时微调 WXSS 样式。
4. **记录归档**：将优化前后的对比与原因记录于本文档中。

### 1. 首页 (Index)

**截图参考**：`1.png` (中文) vs `7.png` (英文) & `2.png` (中文分析中) vs `8.png` (英文分析中)
**状态**：✅ 已完成 (2026-03-31)

#### 视觉与排版评估

- **排版稳定性**：英文版排版稳定，未出现文字溢出或破坏卡片布局的情况。
- **字号与间距**：现有的 CSS 样式自适应良好。
- **多余符号移除**：分析中的提示语前方多余的列表点符号（`•`）在居中排版下显得突兀，中英文均已移除该符号。

#### 文案地道性优化

重点增强文案的“行动感 (Call to Action)”与地道性。

| 位置       | 原英文翻译                                                     | 优化后翻译                                                               | 优化原因                                                          |
| -------- | --------------------------------------------------------- | ------------------------------------------------------------------- | ------------------------------------------------------------- |
| 底部 Tab 栏 | My                                                        | **Profile**                                                         | 在英文 App 规范中，很少单独使用 "My"，"Profile" 或 "Me" 更符合直觉。               |

### 2. 结果页 (Result)

**截图参考**：`9.png` (中文) vs `3.png` & `4.png` (英文)
**状态**：✅ 已完成 (2026-03-31)

#### 视觉与排版评估

1. **食材特征模块 (Taste/Texture/Similar) 排版错乱**：
   - *问题现象*：在中文版中，“味道：”、“口感：”、“类似：”与右侧内容对齐良好。但在英文版中（如 `Taste :`），由于英文单词长度不一，加上冒号被单独挤到了下一行，导致左侧 Label 和右侧 Value 布局极度混乱。
   - *优化方案*：
     - 去除 WXML 中硬编码的中文全角冒号 `：`。
     - 增加 `.feature-label` 的固定宽度（或改为相对比例），确保足够容纳英文单词（如 "Texture"）。
     - 重新调整 Flex 布局，确保左右两侧内容顶部对齐，且右侧长文本正常折行。
2. **底部操作栏按钮 (List to Buy)**：
   - *问题现象*：目前宽度和布局尚可，但英文文案可以进一步优化以提升点击感。

#### 文案地道性优化

| 位置    | 原英文翻译            | 优化后翻译               | 优化原因                                                            |
| ----- | ---------------- | ------------------- | --------------------------------------------------------------- |
| 页面标题  | Report           | **Analysis**        | Report 过于正式（像体检报告），Analysis 更符合 AI 分析结果的语境。                     |
| 鲜度标题  | Freshness Report | **Freshness Check** | 同上，Check 更轻量、口语化。                                               |
| 鲜度依据  | Evidence         | **Indicators**      | Evidence 偏法庭/学术用语，Indicators（关键指标）在消费级 App 中更自然。                |
| 生成采购单 | List to Buy      | **Shopping List**   | "List to Buy" 稍显生硬，"Shopping List" 是标准的电商/菜谱 App 用法，且动词开头更有行动感。 |

### 3. 采购单页 (List)

**截图参考**：`10.png` (中文) vs `5.png` (英文)
**状态**：✅ 已完成 (2026-03-31)

#### 视觉与排版评估

1. **顶部菜谱标签折行**：
   - *问题现象*：在中文版中，“清蒸小鲍鱼”可以完美放在一行。但在英文版中（如 `Rose Myrtle Jam`），由于长度增加，导致右侧的菜谱标签发生内部折行，甚至溢出了标签背景的灰色区域。
   - *优化方案*：
     - 给顶部标题区增加弹性布局保护，当左侧食材名过长时，允许其折行。
     - 给右侧的菜谱标签（`.recipe-badge`）设置最大宽度并允许内部文字正常折行且不溢出背景，或者考虑如果过长直接使用省略号 `text-overflow: ellipsis`，确保 UI 不崩坏。
2. **副标题对齐**：
   - *问题现象*：`Ingredients to prepare` 和 `Check as you buy` 这两句在英文下虽然没有完全错位，但由于英文太长，两者几乎撞到了一起，没有了中文版那种优雅的留白。
   - *优化方案*：微调两者的字体大小，或者调整两者的间距。

#### 文案地道性优化

| 位置   | 原英文翻译                  | 优化后翻译                  | 优化原因                                                         |
| ---- | ---------------------- | ---------------------- | ------------------------------------------------------------ |
| 页面标题 | List to Buy            | **Shopping List**      | 最标准的电商/生活应用术语，无需造词。                                          |
| 副标题左 | Ingredients to prepare | **What you'll need**   | "Ingredients to prepare" 过于直译，"What you'll need"（你需要准备的）更自然。 |
| 副标题右 | Check as you buy       | **Tick off as you go** | "Tick off" (勾选) 比 "Check" 更符合复选框的动作。                         |

### 4. 我的页 (My)

**截图参考**：`11.png` (中文) vs `6.png` (英文)
**状态**：✅ 已完成 (2026-03-31)

#### 视觉与排版评估

1. **历史记录卡片内部折行**：
   - *问题现象*：在英文版中，由于左侧食材名过长（如 "Rose Myrtle Fruit"），且右侧菜谱名也很长，两者挤在一起导致右侧菜谱名发生了不优雅的折行，甚至与时间戳重叠或过于拥挤。
   - *优化方案*：
     - 给 `.history-item` 增加合理的 Flex 布局约束。限制左侧名称（`.history-name`）的最大宽度，超出部分显示省略号。
     - 确保右侧内容（`.history-recipe`）保持独立区域或限制最大行数。

#### 文案地道性优化

| 位置     | 原英文翻译      | 优化后翻译            | 优化原因                                                             |
| ------ | ---------- | ---------------- | ---------------------------------------------------------------- |
| 页面标题   | My Records | **Profile**      | 底部 Tab 已经改为了 Profile，顶部 NavigationBar 保持一致更专业。                   |
| 历史记录标题 | History    | **Recent Scans** | "History" 显得比较干瘪，"Recent Scans"（最近扫描）不仅更生动，还能唤起用户对核心功能（拍照扫描）的记忆。 |
