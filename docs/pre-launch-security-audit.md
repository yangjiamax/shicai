# 识为鲜(PinFresh) - 上线前安全审核清单

> **创建日期**：2026-04-02
> **目的**：正式上线前进行全面的安全排查，确保系统、数据、API和用户隐私的安全。

## 1. API 密钥与敏感信息存储
- [ ] **API 密钥不在前端硬编码**：确保小程序端（如 `miniprogram/` 下的任何文件）未硬编码大模型 API Key（如 OpenAI、千问等）、B站/小红书等第三方服务的 Secret。
- [ ] **环境变量管理**：检查云函数 `analyze` 和 `login` 中使用的 API 密钥是否已通过云开发环境的环境变量（或密钥管理系统）进行配置。
- [ ] **源码脱敏**：检查代码库中是否意外提交了包含真实敏感信息的 `config.js`、`app.js` 或类似配置文件（确保 `.gitignore` 包含相关文件）。

## 2. 云函数安全与权限控制
- [ ] **云函数调用鉴权**：确保云函数仅允许小程序端合法调用，防止被外部恶意抓包请求（验证 `OPENID` 和 `APPID`）。
- [ ] **参数校验与防注入**：云函数（如 `analyze`）对前端传入的 `imageBase64` 等参数必须进行格式、大小限制及安全校验，防止恶意载荷（如超大图片导致 OOM 或内存溢出）。
- [ ] **最小权限原则**：云开发环境中的数据库（如存储历史记录、反馈）、云存储的读写权限应严格按照「仅创建者可读写」或相应的最小权限配置。

## 3. 用户隐私与数据合规
- [ ] **隐私保护指引**：在微信公众平台完善并发布《小程序用户隐私保护指引》，明确说明相机权限、相册权限、用户信息的使用目的。
- [ ] **权限获取规范**：小程序获取摄像头（拍照）、相册（上传）、本地存储等权限时，必须在用户触发相关操作时才进行请求，并做好拒绝授权的优雅降级。
- [ ] **用户数据收集**：确保未过度收集与核心业务无关的用户信息（目前设计为静默登录，未强制获取头像昵称，符合最小必要原则）。
- [ ] **反馈功能防注入**：意见反馈页面（`feedback/`）用户输入的文本内容需进行过滤或转义，防止跨站脚本攻击（XSS）。

## 4. 网络安全与通信
- [ ] **HTTPS 强制与合法域名**：如果小程序内有除了云函数以外的外部请求（`wx.request`），需确保在微信公众平台后台已配置业务合法域名。
- [ ] **防刷与限流策略**：云函数 `analyze` 涉及调用大语言模型（成本较高），需针对单个 `userId` 或 `openid` 设置每日调用频次限制（Rate Limit），防止恶意刷单导致 API 计费失控。

## 5. 内容安全 (UGC)
- [ ] **图片合规检测**：用户上传或拍照的图片在处理或存储前，建议调用微信原生的 `security.imgSecCheck` 接口进行鉴黄、暴恐等违规检测。
- [ ] **文本合规检测**：用户填写的意见反馈内容，建议调用微信原生 `security.msgSecCheck` 接口进行违规词过滤，防止产生和存储违规内容。

## 6. 其他配置检查
- [ ] **关闭调试模式**：确保线上正式版本未开启 `vConsole`，且排查清除了代码中打印敏感信息的 `console.log`。
- [ ] **第三方依赖安全**：检查 `package.json` 中的 npm 包（特别是云函数端）是否存在已知的严重高危漏洞（可运行 `npm audit` 检查）。
